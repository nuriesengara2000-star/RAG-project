Проект RAG (Retrieval-Augmented Generation) — это подход к созданию интеллектуальных приложений, 
которые умеют отвечать на вопросы на основе конкретных документов, а не только на основе 
обученных знаний языковой модели.

Основная идея RAG заключается в объединении двух этапов: поиска информации и генерации ответа.
Сначала система ищет релевантные фрагменты текста в базе данных, а затем передает их языковой 
модели, которая формирует итоговый ответ.

Для хранения документов в RAG-приложениях часто используются векторные базы данных.
Каждый текстовый фрагмент преобразуется в числовой вектор, называемый эмбеддингом.
Эмбеддинги позволяют находить похожие тексты по смыслу, а не по точному совпадению слов.

Supabase — это облачная платформа, построенная поверх PostgreSQL.
Она предоставляет удобный интерфейс для работы с базой данных, а также поддержку расширений,
таких как pgvector, которое позволяет хранить и сравнивать векторы.

Расширение pgvector добавляет новый тип данных vector.
Этот тип используется для хранения эмбеддингов фиксированной размерности, например 1536.
Также pgvector поддерживает различные метрики расстояния, включая косинусное сходство.

В типичном RAG-приложении процесс загрузки данных состоит из нескольких этапов.
Сначала документ считывается из файла.
Затем текст разбивается на небольшие смысловые фрагменты, которые называются чанками.
После этого каждый чанк преобразуется в эмбеддинг с помощью модели машинного обучения.

Когда пользователь задает вопрос, система также преобразует этот вопрос в эмбеддинг.
Затем выполняется семантический поиск — поиск тех чанков документа, которые наиболее близки
к вопросу по смыслу.

Найденные фрагменты текста передаются в языковую модель вместе с вопросом пользователя.
Модель формирует ответ, опираясь только на предоставленный контекст.
Это позволяет снизить количество галлюцинаций и повысить точность ответов.

Таким образом, RAG-приложения широко применяются для работы с документацией,
учебными материалами, корпоративными базами знаний и другими текстовыми источниками.
